# Wisconsin App Privacy Data Inventory - 2026-07-11

## Scope
- App Store target: `Wisconsin` only. `WisconsinKiosk` is not distributed through the App Store.
- Inventory boundary: data sent by the native app to Wisconsin Creative services or third-party services, data returned for authenticated app functionality, protected-device permissions, native SDKs, and server diagnostics generated by native API traffic.
- Source of truth: native Swift/project files, `prisma/schema.prisma`, server instrumentation, production dependency declarations, privacy policy, and `PrivacyInfo.xcprivacy`.

## Explicit Non-Collection For This Build
- No precise or coarse location.
- No Contacts data.
- No payment, financial, health, fitness, browsing-history, search-history, advertising, or cross-app tracking data.
- No camera frames, photos, videos, or audio uploaded by the main iOS target.
- No native crash reports, device-performance metrics, session replay, or generic tap/click analytics.
- No third-party/social login. The app uses the organization's first-party invitation and password system.

## Service Providers And Sharing
- Neon/Postgres and Prisma support first-party account and operational storage.
- Vercel hosts first-party web/API execution and platform logs.
- Sentry is configured on the Next.js web/server service for error and trace diagnostics, including errors generated while serving native API requests; it is not embedded in the native iOS target.
- Apple APNs and ActivityKit deliver notifications and Live Activities.
- Resend supports service email; Upstash supports rate limiting; Vercel Blob serves stored media used by the service.
- No provider receives data for advertising, data brokerage, or cross-app tracking.

## App Store Connect Answers
- Data used to track users: **No**.
- Data linked to identity: **Yes** for every declared type above.
- Data used for third-party advertising, developer advertising/marketing, or other purposes: **No**.
- App Functionality: **Yes** for every declared type above.
- Analytics: **No** for the native App Store target. Server reliability traces are classified as Other Diagnostic Data used for App Functionality.

## Reverification Triggers
- Adding a native crash/analytics/replay SDK.
- Uploading profile images, booking media, audio, or camera frames from the main iOS target.
- Adding location, Contacts, Photos, microphone, advertising, ATT, or third-party login.
- Changing Sentry PII/replay settings or adding user identity to Sentry events.
- Changing account deletion, record retention, notification-token, or server-log behavior.
